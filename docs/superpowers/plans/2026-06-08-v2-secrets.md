# v1 Module #2 — `secrets` Implementation Plan

**Goal:** Backend-agnostic secret resolution + masking, with an env-variable default and a bridge into core's `${secret:...}` placeholder expansion. Fully unit-tested, no external services required.

**Architecture:** `SecretResolver` SPI (ServiceLoader) + `EnvSecretResolver` default; `Secrets` facade (parse `provider:key`, cache, fail-fast, register values for masking); `SecretMasker` scrubs secret values from text; `Secrets.placeholderResolver()` adapts to core's `ValueResolver` so config `${secret:KEY}` expands at load.

**Tech stack:** Java 17, Maven (module under parent), depends on `core`; JUnit 5 + AssertJ tests.

**Status:** ✅ implemented. Run `mvn -q -pl secrets test` (or root `mvn -q test`).

## Contracts

```java
interface SecretResolver { String name(); Optional<String> resolve(String key); }      // SPI

final class Secrets {
    Secrets(List<SecretResolver> resolvers, String defaultProvider, SecretMasker masker); // unknown default → FrameworkException
    String get(String reference);            // "KEY" or "provider:KEY"; cached; missing/unknown → SecretResolutionException
    ValueResolver placeholderResolver();     // bridge: handles ${secret:KEY} in core config
    SecretMasker masker();
}

final class SecretMasker { void register(String secret); String mask(String text); int size(); }
```

## Behaviour verified by tests
- env resolver: name "env"; resolves present key; empty when absent.
- masker: replaces registered secret; leaves clean text; ignores null/blank; null-safe mask.
- facade: default + `provider:` resolution; resolved value auto-masked; missing key fail-fast; unknown provider fail-fast; unknown default provider rejected at construction; `${secret:KEY}` expands through core's `PlaceholderResolver`.
